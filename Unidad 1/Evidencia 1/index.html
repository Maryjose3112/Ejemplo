Ejercicio 1: Cracion y Manipulación de Arreglos

• a) Crea un arreglo vacío y agrega 10 elementos numéricos aleatorios entre 1 y 100.
• b) Imprime el arreglo.
• c) Ordena el arreglo de menor a mayor utilizando el método sort().
• d) Imprime el arreglo ordenado.
• e) Elimina los 3 primeros elementos del arreglo.
• f) Agrega 5 nuevos elementos al final del arreglo.
• g) Imprime el arreglo final.

EXPLICACION CODIGO:

a) Se creo un arreglo llamado arregloprincipal de 10 elementos numéricos. Array.from({ length: 10 }) genera un arreglo de 10 posiciones vacías. El segundo parámetro de Array.from es una función de flecha () =>, que genera un número aleatorio entre 1 y 100 para cada posición del arreglo:
* Math.random() genera un número decimal entre 0 y 1.
* Math.random() * 100 lo convierte en un número entre 0 y 99.
* Math.floor() redondea hacia abajo para obtener un número entero.
* Sumamos +1 para obtener un rango entre 1 y 100.
b) Por medio de un console.log() se imprime el arreglo principal con los 10 valores generado aleatoriamente.
c) Ordena el arreglo arregloprincipal de menor a mayor y guarda el resultado en la variable arregloordenado.sort((a, b) => a - b) es una función de comparación que le dice a sort cómo ordenar los números. Si a - b es negativo, a va antes que b; si es positivo, b va antes que a.
d) Igual que el anterior imprime el arreglo generado aleatoriamente pero ahora de forma ordenada.
e) Elimina los primeros 3 elementos del arreglo arregloordenado usando splice(0, 3), que significa "comienza en el índice 0 y elimina 3 elementos". agrega 5 nuevos elementos al final del arreglo arregloordenado:
* Array.from({ length: 5 }) crea un nuevo arreglo de 5 elementos.
* Cada nuevo número se genera de manera aleatoria entre 1 y 100.
* push(...nuevoArreglo) inserta los nuevos 5 números al final del arreglo arregloordenado.
f) Y por ultimo se imprime el nuevo arreglo creado que se basa principalmente en eliminar los primero tres valores del arreglo ordenado y se le agregan cinco valores nuevos aleatoriamente.

El objeto ahora tiene cuatro propiedades: nombre, edad, ciudad y hobbies. Ahora se imprimirá con las nuevas propiedades agregadas.
c) Luego en la línea siguiente se utiliza el método delate en el cual se elimina la propiedad edad del objeto persona. Después de ejecutar este código, el objeto ya no tendrá la propiedad edad, por lo que dejará de existir en el objeto.

Ejercicio 3: Referencias y Copias

• a) Crea dos variables, a y b, y asígnales el mismo objeto.
• b) Modifica una propiedad del objeto a través de a.
• c) Imprime ambos objetos. ¿Qué observas?
• d) Explica por qué se produce este comportamiento y cómo evitarlo.

EXPLICACION CODIGO:

a) Se crea una variable a y se le asigna un objeto con dos propiedades:
o nombre: una cadena con el valor "Aurora".
o edad: un número con el valor 37.
En lugar de crear un nuevo objeto para b, se hace que b apunte al mismo objeto que a. Esto significa que ambas variables a y b ahora apuntan al mismo objeto en memoria. Si se modifica el objeto usando a o b, ambos "verán" los mismos cambios porque están referenciando el mismo objeto.
b) En este caso, se modifica la propiedad edad del objeto usando la variable a. Como b también está apuntando al mismo objeto, esta modificación también será visible cuando se acceda a través de b.
c) Esto imprime el contenido actual del objeto referenciado por a. Como a ha sido usado para modificar la propiedad edad. Como b apunta al mismo objeto que a, también se verá reflejada la misma modificación en b.
Este comportamiento se debe a que en JavaScript los objetos son referenciados. Esto significa que cuando asignas un objeto a una variable, lo que realmente se almacena en la variable es una referencia (una dirección de memoria) que apunta al objeto, no una copia del objeto en sí. Tanto a como b apuntan al mismo objeto en memoria. Modificar el objeto a través de a (cambiando la propiedad edad a 35) afecta tanto a “a” como a b porque ambos comparten el mismo objeto. El resultado en consola mostrará el mismo objeto para a y b porque no se trata de dos objetos diferentes, sino del mismo objeto referenciado por ambas variables.

Ejercicio 4: Clases

• a) Crea una clase llamada Nodo, que tenga 3 propiedades:
* back
* data
* next
• b) Crea tres variables llamadas nodo1, nodo2, nodo3 y en cada variable realiza una instancia de Nodo.
• c) Asigna un valor a la propiedad data de cada una las variables.
• d) A la propiedad next de nodo1 asígnale como valor la variable nodo2 y a la propiedad next de nodo2 asígnale como valor la variable nodo3, a la propiedad next de nodo3 asígnale el valor de null.
• e) A la propiedad back de nodo1 asígnale el valor de null, a la propiedad back de nodo2 asígnale la variable nodo1. A la propiedad back de nodo3 asígnale como valor la variable nodo2.
• g) Imprime las tres variables y explica que está sucediendo.

EXPLICACION CODIGO:

a) Se crea una clase que se utiliza para crear nodos que serán parte de una lista enlazada. Cada nodo tiene:
* data: Almacena el valor o dato del nodo.
* next: Un puntero (referencia) al siguiente nodo en la lista.
* back: Un puntero (referencia) al nodo anterior en la lista.
b) se crean tres instancias de la clase Nodo, cada una con un valor diferente. Aunque los nodos ya se inicializaron con valores en el constructor, este código vuelve a asignar esos mismos valores. Este paso es innecesario ya que los valores ya se establecieron al crear los nodos.
c) Estableciendo las referencias next:
* nodo1.next = nodo2;: Esto significa que el siguiente nodo después de nodo1 es nodo2.
* nodo2.next = nodo3;: Esto significa que el siguiente nodo después de nodo2 es nodo3.
* nodo3.next = null;: Como nodo3 es el último nodo de la lista, no tiene un siguiente nodo, por lo que se establece su next en null. Estableciendo las referencias back:
* nodo1.back = null;: nodo1 es el primer nodo, por lo que no hay nodo anterior, así que su back es null.
* nodo2.back = nodo1;: nodo2 tiene como nodo anterior a nodo1.
* nodo3.back = nodo2;: nodo3 tiene como nodo anterior a nodo2
d) Este código imprime en la consola la información de cada nodo. Al imprimir un nodo, se mostrará su data, next, y back. El código proporciona una base para gestionar una lista doblemente enlazada, lo que permite añadir, eliminar o recorrer nodos de manera eficiente en ambas direcciones.

Ejercicio 5: Filtrado y Mapeo de Arreglos

• a) Crea un arreglo de 10 objetos que representen a productos con propiedades como nombre, precio y categoría.
• b) Utiliza los métodos filter() y map() para crear un nuevo arreglo que contenga solo los productos de una categoría específica y con un precio mayor a 100.
• c) Explica cómo funcionan estos métodos y cuál es su utilidad.

EXPLICACION CODIGO:

a) Se crea un arreglo llamado productos, donde cada elemento es un objeto con tres propiedades:
* nombre: el nombre del producto.
* precio: el precio del producto.
* categoría: la categoría a la que pertenece el producto.
b) Se utilizan los métodos filter () y map () para crear un nuevo arreglo basado en ciertas condiciones:
* filter() toma una función que verifica una condición para cada elemento del arreglo productos. La condición en este caso es que el producto debe pertenecer a la categoría "Electrónica" y tener un precio mayor a 100.Solo los productos que cumplan ambas condiciones pasarán al siguiente paso.
* Map () se utiliza para crear un nuevo arreglo, transformando los objetos filtrados. En este caso, solo se extraen y devuelven dos propiedades del producto: nombre y precio. Así, el nuevo arreglo contendrá objetos con solo esas dos propiedades, omitiendo la categoría.
c) Este console.log imprimirá el resultado del filtro y mapeo, que será un arreglo con los productos de la categoría "Electrónica" y con un precio mayor a 100.

Ejercicio 6: Validacion

• a) Escribe una función que divida dos números.
• b) Maneja la excepción que se produce cuando se intenta dividir por cero.

EXPLICACION CODIGO:

a) La función dividir toma dos parámetros, a y b, que serán los números a dividir (donde a es el dividendo y b es el divisor).
b) Se verifica si alguno de los dos valores (a o b) no es de tipo numérico usando el operador typeof. Si alguno no es un número, se lanza un error con el mensaje: "Ambos valores deben ser numéricos.".
c) Luego, se verifica si el valor de b (el divisor) es igual a 0.
d) Si b es 0, se lanza un error con el mensaje: "No se puede dividir por cero.", ya que la división por cero es indefinida o no permitida en la mayoría de los lenguajes de programación.
e) Si los valores son válidos y no se ha lanzado ningún error, se realiza la división de a entre b y se retorna el resultado.
